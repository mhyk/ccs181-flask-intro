Setting Up Flask

Installing Flask

  • Requires Python 3.8+
  • Install with pip install flask
  • Verify installation:

python -m flask --version
Always confirm Python version is compatible.

Managing Environments

  • Why use virtual environments?
    • Isolate dependencies per project
    • Prevent conflicts between libraries
    • Keep global Python clean
  • Options:
    • venv (built-in)
    • pipenv (dependencies + lock file)
    • pyenv (manage multiple Python versions)

Using venv


# Create environment
python -m venv venv

# Activate (Linux/Mac)
source venv/bin/activate

# Activate (Windows)
venv\Scripts\activate

# Install Flask
pip install flask

Using pipenv


# Install pipenv
pip install pipenv

# Create env and install Flask
pipenv install flask

# Activate shell
pipenv shell

# Install dev packages
pipenv install pytest --dev

Using pyenv

  • Manages multiple Python versions.
  • Useful when working with projects that need different versions.

# Install a new Python version
pyenv install 3.11.9

# Set local project version
pyenv local 3.11.9

# Check active version
pyenv versions

Requirements Management

  • With venv ‚Üí freeze dependencies:

pip freeze > requirements.txt
  • With pipenv ‚Üí Pipfile + Pipfile.lock
  • Good practice: commit these files to version control.

HTML Escaping

  • Flask escapes HTML by default
  • Prevents XSS attacks

Running Flask Applications

  • Flask apps can be started in multiple ways:
    1. Using flask run (recommended)
    2. Running python app.py (with app.run())
    3. Using WSGI servers (production)
  • Environment variables control behavior:
    • FLASK_APP ‚Äì entry point file/module
    • FLASK_ENV ‚Äì development or production
    • FLASK_RUN_HOST ‚Äì bind address (default: 127.0.0.1)
    • FLASK_RUN_PORT ‚Äì port number (default: 5000)

Application Discovery

  • Flask automatically discovers your application using:
    1. FLASK_APP environment variable
    2. Common patterns if FLASK_APP not set:
      • wsgi.py (looks for app or application)
      • app.py (looks for app or application)

# app.py - Flask will find this automatically
from flask import Flask

app = Flask(__name__)  # Variable name matters!

@app.route('/')
def hello():
    return 'Hello, World!'

Using python-dotenv

  • Install the python-dotenv package:

$ pip install python-dotenv
  • Create a .env file in your project root:

# .env
FLASK_APP=app.py
FLASK_ENV=development
FLASK_RUN_HOST=0.0.0.0
FLASK_RUN_PORT=8000
DATABASE_URL=sqlite:///app.db
SECRET_KEY=your-secret-key-here
  • Flask automatically loads .env files when python-dotenv is installed

Environment File Hierarchy

  • Flask loads environment files in this order:
    1. .flaskenv ‚Äì Flask-specific settings (committed to git)
    2. .env ‚Äì Sensitive/local settings (add to .gitignore)

# .flaskenv (version controlled)
FLASK_APP=app.py
FLASK_ENV=development

# .env (local/sensitive - not version controlled)
PIPENV_VENV_IN_PROJECT=1
SECRET_KEY=super-secret-key
DATABASE_URL=postgresql://user:pass@localhost/db
  • Variables in .env override those in .flaskenv
Always add .env to your .gitignore file!

Routing in Flask

What is Routing?

  • Routing is the process of matching a URL path to a function in your web application.
  • Each route corresponds to a view function that handles the request.
  • Flask uses the @app.route() decorator to define routes.
  • Example: /hello ‚Üí function that returns "Hello, World".
Explain that routes define how users access resources in the web app.

Basic Routing Example


from flask import Flask
app = Flask(__name__)

@app.route("/")
def index():
    return "This is the homepage."

@app.route("/hello")
def hello():
    return "Hello, Flask!"
Point out that the decorator binds URL paths to Python functions.

Dynamic Routes


@app.route("/user/<username>")
def show_user(username):
    return f"Hello, {username}!"

@app.route("/post/<int:post_id>")
def show_post(post_id):
    return f"Post number {post_id}"
  • Use <variable> in routes to capture URL parameters.
  • Type converters: int, float, path.
Dynamic routes make pages flexible, e.g. user profiles.

Templating with Jinja2

What is a Templating Engine?

  • A system that combines static HTML with dynamic data.
  • Separates presentation (HTML) from logic (Python).
  • Prevents repetitive code (DRY principle).
  • Examples: Jinja2 (Flask), Django Templates, Mustache, Handlebars.

What is Jinja2?

  • Flask‚Äôs default templating engine.
  • Allows variables, loops, conditions, and filters inside HTML.
  • Syntax:
    • {{ ... }} ‚Üí output a variable or expression
    • {% ... %} ‚Üí control statements (if, for, block)
    • {# ... #} ‚Üí comments

Rendering Templates


from flask import render_template

@app.route("/")
def index():
    return render_template("index.html")
  • Templates go in templates/ folder
  • Use Jinja2 syntax

Sessions and Flashing

Sessions in Flask

  • HTTP is stateless: each request is independent.
  • A session lets us store data across requests.
  • Flask sessions:
    • Stored on the client as a secure cookie.
    • Signed with your app‚Äôs SECRET_KEY.
    • Prevents tampering (but data is visible to the client).
  • Common use cases:
    • User authentication (e.g., logged-in user ID).
    • Temporary preferences (e.g., theme, language).
    • Shopping carts or workflow progress.

Takeaway

Sessions give ‚Äúmemory‚Äù to your web app while still running on stateless HTTP.

Message Flashing

  • A way to send temporary messages from the server to the user.
  • Typically used for:
    • Form validation feedback.
    • Success / error notifications.
    • Login / logout confirmations.
  • Messages are stored in the session.
  • They disappear after being displayed once.

Takeaway

Flashing is Flask‚Äôs way of showing ‚Äúone-time‚Äù feedback to users.

Middleware in Flask

What is Middleware?

A layer between the web server and your Flask app. It can inspect or modify requests and responses.

Why Use It?

  • Logging or monitoring every request
  • Authentication / authorization checks
  • Adding security headers
  • Compression (e.g., gzip responses)

How It Works

  1. Flask is a WSGI application
  2. Middleware wraps the Flask app
  3. Requests pass ‚Üí Middleware ‚Üí Flask ‚Üí Response back through Middleware

Takeaway

Middleware = reusable layers that sit ‚Äúin front of‚Äù or ‚Äúaround‚Äù your Flask app.
